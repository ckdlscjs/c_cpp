# 🛠️ DX11 기반 ECS 아키처 캐시 최적화 렌더러 (c_cpp/PardCode34)

본 프로젝트는 DirectX 11 API와 C++17 표준을 활용하여 캐시 성능 및 렌더링 파이프라인 바인딩을 최적화한 3D 그래픽스 렌더러 엔진 프로젝트임.

학습 및 설계의 기반으로 **이득우의 게임수학**, **DirectX 11 물방울책 (Introduction to 3D Game Programming with Direct3D 11 - Frank D. Luna 저)**, **RasterTek DirectX 11 Tutorial**, **PardCode DirectX 11 Tutorial**의 핵심 그래픽스 및 선형대수 인프라를 바닥부터 심도 있게 학습하여 내재화하였으며, 과거 팀 프로젝트인 **맵툴(MapTool)** 및 **게임프로젝트(LoLShinSouls)** 개발 과정에서 누적된 3D 기하 충돌 제어 및 카메라 기하 연출 노하우를 녹여내어 한 단계 진화된 최적화 렌더패스 엔진을 직접 설계 및 구축함.

---

## 00 [ 프로젝트 아키텍처 개요 ]

### 1. 개발 배경 및 기술적 기반
기성 그래픽스 교과서 및 튜토리얼을 통해 D3D11 디바이스 및 자원 라이프사이클의 기저를 이해한 뒤, 단순히 리소스를 렌더링하는 것을 넘어 실제 대형 월드 스폰 시 발생하는 **CPU-GPU 레지스터 적재 병목, 힙 파편화로 인한 메모리 캐시 레이턴시, 잦은 파이프라인 바인딩 오버헤드**를 제어하기 위한 구조적 해법을 고민함.
이에 따라 기저 디바이스 바인딩 모델에 머무르지 않고, 객체지향 렌더링의 한계를 탈피하기 위한 **DOD 기반 ECS 아키텍처**와 **64비트 정렬 키 기반의 다중 렌더패스 스케줄러**를 독자 구현하여 렌더러 파이프라인 전반을 관통하는 최적화를 구현함.

## 01 [ 엔진 아키텍처 및 기반 시스템 ]

### 1. CRTP 기반 시스템 아키텍처 및 RAII 자원 제어
* **기술적 오버뷰**:
  - `EngineSystem`, `ResourceSystem` 등 전역에서 유일성이 보장되어야 하는 매니저 클래스들의 싱글톤 보일러플레이트 코드를 줄이기 위해 C++ CRTP(Curiously Recurring Template Pattern) 패턴을 도입한 `BaseSystem<T>` 기저를 구축함.
  - 이를 통해 인스턴스가 실제 기동되는 최초 시점에 비동기 안전하게 로드되는 지연 생성(Lazy Initialization) 구조를 구현하였으며, 리소스 래퍼 클래스들에 RAII 패턴을 준수하게 하여 디바이스 반환 시점의 자원 생명주기를 일원화하고 누수를 원천 차단함.

### 2. 수학 라이브러리 정의 (Vector, Matrix, Quaternion)
* **기술적 오버뷰**:
  - 수학 클래스들의 멤버변수는 DirectXMath의 SIMD 구조체(`XMVECTOR`, `XMMATRIX`)를 사용하되, 이를 연산하는 함수들과 오버로딩 연산자 등을 구성해 사용하는 래퍼 클래스에 해당하는 Vector2~4, Matrix4x4, Quaternion 클래스를 구현함.
  - 16바이트 정렬 조건(`__declspec(align(16))`)을 적용해 SIMD 레지스터 적재 효율을 보존하고, `union`을 사용해 행렬 구조체 내부에 `XMMATRIX`, `Vector4`를 공용체로 구성해 첨자 접근으로 디버깅 및 수정이 가능하도록 구성함.
  - 오일러 각 이용의 한계인 짐벌락 해결을 위해 사원수 클래스를 구성하고 사원수와 벡터, 행렬 등의 복합 연산을 구성하여 렌더러 전역에서 핵심 연산 자료구조로 활용함.

## 02 [ 데이터지향 설계(DOD) 기반 ECS 아키텍쳐 ]

### 1. ECS 컴포넌트 선형 배열화 및 Swap & Pop 방식
* **기술적 오버뷰**:
  - 기존 OOP 기반의 동적 오브젝트 구조가 초래하는 메모리 파편화를 극복하기 위해 데이터지향 설계(DOD)로 패러다임을 전환함. 컴포넌트 데이터를 16KB 크기의 물리 연속 공간 배열인 컴포넌트 청크(`ChunkData<T>`)로 구성하여 CPU 캐시 적중률(Cache Locality)을 극한으로 조율함.
  - 엔티티 삭제 시 발생하는 메모리 복사 시프트($O(N)$) 비용을 완화하기 위해, 삭제 대상 컴포넌트를 청크 최후방 데이터와 1대1 위치 스왑(`Swap`)한 후 마지막 인덱스를 단순히 `Pop` 처리하는 Swap & Pop 기법을 구현하여 O(1) 고속 소멸을 구현함.

### 2. Sparse/Dense 이중 룩업 테이블 역추적
* **기술적 오버뷰**:
  - Swap & Pop 기동 시 실제 메모리 공간 상의 순서가 바뀐 최후방 엔티티의 참조 인덱스가 꼬이는 현상을 스크리닝하기 위해, `m_LookupTable`(Sparse)과 `m_Entitys`(Dense)를 활용한 이중 룩업 역추적 테이블을 가동함.
  - 가변인자 템플릿과 폴드 표현식(`Fold Expression`)을 통해 이종 컴포넌트 조합(`Comps...`)을 컴파일 타임에 언팩 조립함으로써 런타임 캐스팅 오버헤드를 차단함.

## 03 [ 리소스 캐싱 및 64비트 정렬키 기반 렌더 큐 파이프라인 ]

### 1. 64비트 정렬키 기반 렌더패스 최적화 및 상태 스위칭 최소화
* **기술적 오버뷰**:
  - 매 프레임 수천 번의 드로우콜 시 발생하는 셰이더 및 디바이스 상태 전환 비용을 절감하기 위한 렌더 큐 파이프라인을 구축함.
  - `Pass(4bit) - Shader(16bit) - State(8bit) - Resource(16bit) - Distance(20bit)` 로 구성된 64비트 정렬 키(`_RPKey`)를 마스킹 조립하고 렌더 패스 제출 전 정렬하여 상태 변경 빈도를 최소화함.
  - 렌더 큐 순회 시 이전 키와 현재 키 간의 XOR 비트 연산(`keyCur ^ keyPrev`)을 수행하여, 변경이 감지된 파이프라인 단계(RTV/셰이더/상태/버퍼)만 선별적으로 교체 바인딩함. 중복 바인딩 API 호출 횟수를 극적으로 줄임.

## 04 [ 가시성 최적화 및 고급 그래픽스 파이프라인 ]

### 1. 절두체 컬링 & 마우스 피킹 2단계 필터링
* **기술적 오버뷰**:
  - 절두체 평면과의 교차 판정을 활용해 가시 영역 바깥의 불필요한 연산을 선 차단하는 씬 컬링을 구현함.
  - 카메라 뷰-프로젝션 역행렬을 이용해 NDC 코너 정점들을 월드 투영하고 외적하여 6개 절두체 평면 식을 실시간 생성함. 이후 OBB 중심 및 각 면의 축 벡터 투영 거리를 활용하여 비가시 공간(`F_BACK`)으로 분류해 렌더 수집에서 제외함.
  - 마우스 레이 피킹 시 1차로 각 엔티티의 경계 OBB/AABB 박스와 레이 간의 교차 판정을 수행하여 충돌 비대상 객체를 빠르게 선거하고, 충돌 가능성이 감지된 액터의 실제 삼각형 정점 데이터들만 선별하여 2차 정밀 광선 교차 검사로 가장 가까운 피킹 월드 교점을 산출함.

### 2. GPU Compute Shader 기반 정밀 충돌 판정 가속 (Möller-Trumbore)
* **기술적 오버뷰**:
  - 다량의 메시 스키닝 애니메이션이 구동 중인 씬에서 수천 개의 삼각형 정밀 피킹 판정을 CPU가 순차 처리할 때 발생하는 스터터링 병목을 해결하기 위해 GPU 계산 셰이더를 도입함.
  - `numthreads(64, 1, 1)` 기반의 Compute Shader 상에서 정점별 스키닝 연산 처리를 수만 개 스레드로 동시 분배하고, **Möller-Trumbore 알고리즘** 레이-삼각형 교차 판정 공식을 GPU 상에서 병렬 구동함. 연산 결과 거리를 구조화 버퍼(Structured Buffer) 및 UAV로 기록하고 CPU로 복치하는 최적화 파이프라인을 구축함.

## 05 [ 세부 렌더패스별 D3D11 파이프라인 연동 ]

1. **Deferred Pass (GBuffer MRT)**: GBuffer MRT(Position, Normal, Albedo)에 렌더링하고, 최종 후처리 조명 패스에서 이를 읽어 조명 연산을 수행함으로써 조명 연산의 오버헤드를 픽셀 단위로 최소화함.
2. **Skinned Animation Pass**: 행렬 버퍼에 본(Bone) 행렬 리스트를 셋업하고, 정점 셰이더 내에서 4가중치 스키닝 연산을 가동해 기하 형태를 변형 렌더링함.
3. **Shadow Pass와 투영 텍스처링 PCF 기법**: 그림자 깊이 맵 생성 시 RTV 바인딩을 nullptr로 비우고, DepthStencilView(DSV)만 그림자 맵 리소스로 바인딩하여 패스를 갱신함. 픽셀 셰이더 단에서는 3x3 비교 샘플러(PCF)를 가동하여 그림자 경계면을 부드럽게 감쇠함.
4. **Environment Map(CubeMap) Pass**: 큐브맵 패스 탈출 시 밉맵 동적 생성 및 t7 레지스터에 SRV 바인딩을 수행함. 기하 셰이더 내에서 6개 큐브 뷰포트 인덱스를 동적 할당하여 1회 스트림 방출 렌더링을 구현함.
5. **Outline Pass & Debug Pass**: 
  - **Outline Pass**: 외곽선 실루엣 렌더 패스에서, 스키닝 애니메이션된 정점의 법선(Normal) 방향으로 설정된 두께만큼 정점 위치를 외곽으로 확장 팽창 렌더링하여 카툰 외곽선을 출력함.
  - **Debug Pass**: 디버그용 기하 정보를 그리기 위해, 버텍스 버퍼 자원 제출 없이 기하 셰이더 내부에서 최소/최대 바운딩 좌표 조합(8개)을 절차적으로 자동 팽창시켜 AABB/OBB 디버그 박스를 실시간 렌더링함. 구형 디버깅 시 테셀레이션 파이프라인의 Domain Shader 내에서 삼각형 가중치를 적용한 무게중심 보간 점을 구체 반지름(`radius`)으로 투영하여 매끈한 디버그 구형 구체를 버퍼리스 렌더링함.
